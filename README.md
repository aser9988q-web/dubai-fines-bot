# Dubai Fines Bot

نظام ويب للاستعلام عن المخالفات المرورية من موقع شرطة دبي الرسمي. يعتمد التطبيق على خادم Node.js وواجهة React، مع دعم مسار تشغيل مباشر ومسار بديل عبر **Relay/Proxy** عند تعذر وصول بيئة الإنتاج إلى واجهات شرطة دبي مباشرة.

## المميزات

| الميزة | الوصف |
|---|---|
| الاستعلام عن المخالفات | جلب المخالفات من خدمة شرطة دبي الرسمية مع إظهار التفاصيل الأساسية |
| واجهة عربية | تصميم RTL ودعم واجهة استخدام عربية بالكامل |
| دعم الإمارات | اختيار جهة إصدار اللوحة وكودها ورقمها لجميع الإمارات المدعومة |
| سجل الاستعلامات | حفظ عمليات البحث السابقة داخل قاعدة البيانات |
| مسار بديل للإنتاج | دعم Relay أو Proxy لتجاوز فشل الاتصال المباشر من بعض بيئات الاستضافة |

## التقنيات المستخدمة

| الطبقة | التقنية |
|---|---|
| الواجهة | React 19 + Tailwind CSS 4 + shadcn/ui |
| الخادم | Node.js + Express + tRPC |
| قاعدة البيانات | MySQL عبر Drizzle ORM |
| اللغة | TypeScript |

## النشر على Railway

### 1. إنشاء مشروع جديد على Railway

اذهب إلى [Railway](https://railway.app)، ثم أنشئ مشروعًا جديدًا من GitHub واختر هذا المستودع.

### 2. إضافة قاعدة بيانات MySQL

أضف خدمة MySQL داخل مشروع Railway، ثم انسخ قيمة `DATABASE_URL` من إعداداتها.

### 3. متغيرات البيئة المطلوبة

| المتغير | الوصف | مطلوب |
|---|---|---|
| `DATABASE_URL` | رابط قاعدة بيانات MySQL | نعم |
| `JWT_SECRET` | مفتاح سري عشوائي لتشفير الجلسات | نعم |
| `NODE_ENV` | اجعل القيمة `production` | نعم |
| `DUBAI_POLICE_RELAY_ENABLED` | تفعيل استخدام relay بدل الاتصال المباشر | عند استخدام relay |
| `DUBAI_POLICE_RELAY_URL` | الرابط الأساسي لخدمة relay، مثل `https://your-relay.example.com` | عند استخدام relay |
| `DUBAI_POLICE_RELAY_TOKEN` | الرمز السري الذي يحمي relay ويجب أن يتطابق بين الخدمتين | عند استخدام relay |
| `DUBAI_POLICE_PROXY_ENABLED` | تفعيل استخدام بروكسي HTTPS عند الحاجة | اختياري |
| `DUBAI_POLICE_PROXY_URL` | رابط البروكسي المستخدم للوصول إلى شرطة دبي | عند استخدام proxy |
| `OUTBOUND_PROXY_URL` | بديل عام للبروكسي إذا رغبت بتوحيد الإعدادات | اختياري |

### 4. أوامر البناء والتشغيل للتطبيق الأساسي

| الغرض | الأمر |
|---|---|
| البناء | `pnpm install && pnpm run build` |
| التشغيل | `pnpm run start` |

### 5. أوامر البناء والتشغيل لخدمة Relay

إذا كانت بيئة التطبيق الأساسية لا تستطيع الوصول إلى `www.dubaipolice.gov.ae` بسبب مشكلة TLS أو قيود شبكة، فقم بتشغيل خدمة Relay منفصلة في بيئة يمكنها الوصول إلى الخدمة الحكومية أو عبر بروكسي صالح.

| الغرض | الأمر |
|---|---|
| بناء relay | `pnpm install && pnpm run build:relay` |
| تشغيل relay | `pnpm run start:relay` |

ينبغي ضبط `DUBAI_POLICE_RELAY_TOKEN` على نفس القيمة في التطبيق الأساسي وخدمة relay. كما يمرر التطبيق الآن `plateCodeId` و`plateCategory` صراحةً إلى relay، حتى لا تضيع دقة اختيار كود اللوحة في بيئات الإنتاج.

## التشغيل المحلي

```bash
pnpm install
pnpm run dev
pnpm test
```

ولتجربة خدمة relay محليًا يمكن استخدام الأمر التالي:

```bash
pnpm run dev:relay
```

## ملاحظة تشغيلية

إذا فشل الاتصال المباشر من الخادم إلى شرطة دبي برسائل TLS أو `ERR_CONNECTION_CLOSED`، فهذا لا يعني أن التطبيق معطّل منطقيًا، بل يعني غالبًا أن بيئة الاستضافة نفسها تحتاج إلى **Relay** أو **Proxy** مستقر. لذلك يُفضَّل في الإنتاج إعداد أحد هذين المسارين بدل الاعتماد على الاتصال المباشر فقط.
